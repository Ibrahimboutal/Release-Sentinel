# Presentation Outline

## Slide 1: Release Sentinel

Claim: Agentic testing needs governed release decisions, not just more test execution.

Proof: One-screen architecture showing change manifest, risk agent, Test Cloud execution, triage, release gate, and Action Center.

## Slide 2: Enterprise Problem

Claim: AI-driven automations change faster than manual test planning can react.

Proof: Three pain points: over-testing wastes time, under-testing misses risk, failure triage delays releases.

## Slide 3: Solution

Claim: Release Sentinel chooses the right tests and explains the release decision.

Proof: Flow from change risk to selected Test Manager sets to verdict.

## Slide 4: UiPath Platform Usage

Claim: UiPath is the orchestration, testing, and governance layer.

Proof: Test Cloud/Test Manager, API Workflows, Agent Builder/coded agent, Action Center, UiPath for Coding Agents.

## Slide 5: Demo Scenario

Claim: ClaimsPilot routing changes are a realistic regulated workflow test.

Proof: Change manifest and coverage map excerpts.

## Slide 6: Failure Triage

Claim: The system distinguishes product bugs from fragile tests and data issues.

Proof: Example failed eligibility assertion and recommended fix.

## Slide 7: Human-In-The-Loop

Claim: Ambiguous cases are routed to people.

Proof: Action Center payload with risk, evidence, and approval choices.

## Slide 8: Impact

Claim: Release Sentinel increases confidence while reducing unnecessary regression.

Proof: Low-risk smoke-only path, high-risk targeted path, critical full-regression fallback.

## Slide 9: What Is Next

Claim: The pattern can scale across enterprise automations.

Proof: Test Manager metadata, flakiness history, change-impact learning, and governance reporting.
